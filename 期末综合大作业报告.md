## 一、项目背景

### 1.1 项目开发目的

随着互联网技术的飞速发展和电子商务的普及，在线购物已成为人们日常生活中不可或缺的一部分。无论是大型电商平台还是小型垂直电商，其核心业务逻辑都涵盖了用户管理、商品管理、购物车和订单处理等模块。作为软件工程专业的学生，掌握Web开发的基本技能，理解电商系统的核心业务流程，是未来参与企业级项目开发的重要基础。

本项目旨在运用本学期《JavaWeb开发技术》课程所学的Servlet、JSP、JDBC、过滤器（Filter）、会话管理（Session）等核心技术，结合MySQL数据库和Maven项目管理工具，从零搭建一个功能完整的简易购物网站——ShoppingWeb。通过该项目，将理论知识与工程实践相结合，加深对MVC分层架构思想的理解，锻炼数据库设计与SQL编写能力，培养完整的软件项目开发流程意识。

### 1.2 相关文献及技术资料综述

在项目开发过程中，参考了以下技术资料：

（3）《MySQL必知必会》（Ben Forta 著），该书简明扼要地介绍了MySQL数据库的常用操作，包括数据表设计、CRUD操作、多表连接查询和事务处理，为项目的数据库层设计提供了指导。

## 二、需求分析

### 2.1 功能需求概述

本系统是一个B/S架构的简易购物网站，用户通过浏览器访问，无需安装客户端。系统包含**用户端**和**管理端**两大功能模块。

#### 2.1.1 用户端功能

1. **用户注册与登录**：新用户通过注册页面填写用户名、密码和邮箱完成注册；已注册用户通过登录页面输入用户名和密码进行身份验证，登录成功后系统将用户信息存入Session会话中。

2. **商品浏览与搜索**：用户可在首页和商品列表页浏览商品，支持按分类筛选（电子产品、服装配饰、图书教材、家居生活、零食食品共五个分类），支持按关键词搜索商品名称和描述，支持分页显示（每页8条）。

3. **商品详情查看**：用户点击商品可进入详情页，查看商品的完整描述、价格、库存等信息。

4. **购物车管理**：登录用户可将商品添加到购物车，在购物车页面修改商品数量或删除商品，系统自动计算每项商品的小计金额。

5. **订单结算**：用户确认购物车商品后提交订单，系统在事务中完成：生成订单记录 → 保存订单明细 → 扣减商品库存 → 清空用户购物车。下单成功后跳转至订单详情页。

6. **订单查看**：登录用户可查看自己的历史订单列表，点击订单可查看详情（含订单明细、总金额、订单状态等）。

#### 2.1.2 管理端功能

1. **商品管理**：管理员可对商品进行增删改查操作（添加新商品、编辑已有商品、删除商品、查看商品列表）。

2. **订单管理**：管理员可查看所有用户的订单列表，查看订单详情，修改订单状态（待付款→已付款→已发货→已完成→已取消）。

#### 2.1.3 非功能需求

1. **权限控制**：普通用户无法访问管理后台，未登录用户无法访问购物车和订单页面。
2. **安全性**：用户密码采用MD5加密存储，数据库操作使用PreparedStatement防止SQL注入。
3. **编码统一**：全局统一使用UTF-8编码，避免中文乱码问题。
4. **事务一致性**：下单操作在数据库事务中完成，保证库存、订单、购物车数据的一致性。

### 2.2 用例分析

- **普通用户角色**：注册账号 → 登录系统 → 浏览商品 → 搜索/筛选商品 → 添加购物车 → 修改购物车 → 提交订单 → 查看订单
- **管理员角色**：登录系统 → 管理商品（增删改查）→ 管理订单（查看、修改状态）

### 4.2 分层架构说明

**（1）实体层（Model）**：定义与数据库表对应的Java类和实体关系。例如User类包含id、username、password、email、role等属性；Order类内部定义了OrderItem内部类表示订单明细。所有实体类均实现Serializable接口。

**（2）数据访问层（DAO）**：封装对数据库的增删改查操作，使用JDBC + PreparedStatement实现。每个DAO类专注于一张或几张关联表的操作。

**（3）控制器层（Servlet）**：处理HTTP请求，调用DAO层完成业务逻辑，将处理结果转发到JSP页面进行展示。每个Servlet覆盖一个功能模块，通过action参数区分不同操作。

**（4）视图层（JSP）**：负责页面渲染，使用JSTL标签库和EL表达式动态展示数据，以header.jsp和footer.jsp作为公共模板复用。

**（5）过滤器（Filter）**：EncodingFilter统一处理请求/响应编码（UTF-8）；LoginFilter拦截需要登录的页面；AdminFilter拦截管理后台请求并验证管理员身份。

**（6）工具类（Utils）**：DBUtil提供数据库连接获取、资源关闭、事务回滚等通用方法，使用Properties配置文件管理数据库连接参数。

### 4.3 核心技术要点

1. **数据库连接管理**：通过静态代码块加载db.properties配置文件和JDBC驱动，每次请求时通过getConnection()获取连接，用完即关，避免连接泄漏。

2. **密码安全**：用户注册和登录时，SQL中使用MySQL内置的MD5()函数对密码进行哈希处理，数据库存储的是密文而非明文。

3. **SQL注入防范**：所有SQL操作均使用PreparedStatement预编译语句，参数通过占位符（?）传入，有效防止SQL注入攻击。

4. **事务处理**：下单操作在一个数据库事务中完成（插入订单→插入明细→扣库存→清空购物车），任一环节失败则整体回滚，保证数据一致性。

5. **会话管理**：用户登录成功后将User对象存入HttpSession，各页面通过session获取当前用户信息，退出时销毁session。

6. **URL路由设计**：利用@WebServlet注解和HttpServletRequest.getPathInfo()方法实现RESTful风格的URL路由。

### 5.1 数据库工具类 DBUtil

```java
package com.shop.utils;

import java.io.InputStream;
import java.sql.*;
import java.util.Properties;

public class DBUtil {

    private static final String DRIVER = "com.mysql.cj.jdbc.Driver";
    private static String dbUrl;
    private static String user;
    private static String password;
    private static boolean driverLoaded = false;

    static {
        try (InputStream in = DBUtil.class.getClassLoader()
                .getResourceAsStream("db.properties")) {
            Properties props = new Properties();
            props.load(in);
            dbUrl    = props.getProperty("db.url");
            user     = props.getProperty("db.username");
            password = props.getProperty("db.password");
        } catch (Exception e) {
            System.err.println("数据库配置加载失败: " + e.getMessage());
        }
        try {
            Class.forName(DRIVER);
            driverLoaded = true;
        } catch (ClassNotFoundException e) {
            System.err.println("MySQL驱动加载失败: " + e.getMessage());
        }
    }

    public static Connection getConnection() throws SQLException {
        return DriverManager.getConnection(dbUrl, user, password);
    }

    public static void close(Connection conn, Statement stmt, ResultSet rs) {
        try { if (rs != null)   rs.close(); } catch (SQLException e) { e.printStackTrace(); }
        try { if (stmt != null) stmt.close(); } catch (SQLException e) { e.printStackTrace(); }
        try { if (conn != null) conn.close(); } catch (SQLException e) { e.printStackTrace(); }
    }

    public static void rollback(Connection conn) {
        if (conn != null) {
            try { conn.rollback(); } catch (SQLException e) { e.printStackTrace(); }
        }
    }
}
```

### 5.2 用户登录功能 LoginServlet

```java
package com.shop.servlet;

import com.shop.dao.UserDao;
import com.shop.model.User;

import javax.servlet.ServletException;
import javax.servlet.annotation.WebServlet;
import javax.servlet.http.*;
import java.io.IOException;

@WebServlet("/login")
public class LoginServlet extends HttpServlet {

    private final UserDao userDao = new UserDao();

    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp)
            throws ServletException, IOException {
        req.getRequestDispatcher("/login.jsp").forward(req, resp);
    }

    @Override
    protected void doPost(HttpServletRequest req, HttpServletResponse resp)
            throws ServletException, IOException {
        String username = req.getParameter("username");
        String password = req.getParameter("password");

        if (username == null || password == null ||
                username.trim().isEmpty() || password.trim().isEmpty()) {
            req.setAttribute("error", "用户名和密码不能为空！");
            req.getRequestDispatcher("/login.jsp").forward(req, resp);
            return;
        }

        User user = userDao.findByUsernameAndPassword(
                username.trim(), password.trim());
        if (user == null) {
            req.setAttribute("error", "用户名或密码错误！");
            req.getRequestDispatcher("/login.jsp").forward(req, resp);
            return;
        }

        HttpSession session = req.getSession();
        session.setAttribute("user", user);
        resp.sendRedirect(req.getContextPath() + "/index.jsp");
    }
}
```

### 5.3 订单结算功能（含事务处理） OrderDao.createOrder()

```java
public int createOrder(Integer userId, String address,
        List<OrderItem> items, BigDecimal totalPrice) {
    Connection conn = null;
    try {
        conn = DBUtil.getConnection();
        conn.setAutoCommit(false);  // 开启事务

        // 1. 插入订单
        String orderNo = "ORD" + System.currentTimeMillis() +
            UUID.randomUUID().toString().substring(0, 6).toUpperCase();
        String sqlOrder = "INSERT INTO orders (order_no, user_id, " +
            "total_price, status, address) VALUES (?, ?, ?, 'pending', ?)";
        int orderId;
        try (PreparedStatement stmt = conn.prepareStatement(
                sqlOrder, Statement.RETURN_GENERATED_KEYS)) {
            stmt.setString(1, orderNo);
            stmt.setInt(2, userId);
            stmt.setBigDecimal(3, totalPrice);
            stmt.setString(4, address);
            stmt.executeUpdate();
            try (ResultSet rs = stmt.getGeneratedKeys()) {
                if (!rs.next()) { conn.rollback(); return -1; }
                orderId = rs.getInt(1);
            }
        }

        // 2. 插入订单明细 + 扣库存
        String sqlItem = "INSERT INTO order_items (order_id, product_id, " +
            "product_name, product_price, quantity) VALUES (?, ?, ?, ?, ?)";
        String sqlStock = "UPDATE products SET stock = stock - ? " +
            "WHERE id = ? AND stock >= ?";
        try (PreparedStatement stmtItem = conn.prepareStatement(sqlItem);
             PreparedStatement stmtStock = conn.prepareStatement(sqlStock)) {
            for (OrderItem item : items) {
                stmtItem.setInt(1, orderId);
                stmtItem.setInt(2, item.getProductId());
                stmtItem.setString(3, item.getProductName());
                stmtItem.setBigDecimal(4, item.getPrice());
                stmtItem.setInt(5, item.getQuantity());
                stmtItem.executeUpdate();

                stmtStock.setInt(1, item.getQuantity());
                stmtStock.setInt(2, item.getProductId());
                stmtStock.setInt(3, item.getQuantity());
                int affected = stmtStock.executeUpdate();
                if (affected == 0) {
                    conn.rollback();
                    return -1;    // 库存不足
                }
            }
        }

        // 3. 清空购物车
        new CartDao().clearByUserId(userId, conn);

        conn.commit();          // 提交事务
        return orderId;
    } catch (SQLException e) {
        DBUtil.rollback(conn);  // 回滚事务
        e.printStackTrace();
    } finally {
        if (conn != null) {
            try { conn.setAutoCommit(true); conn.close(); }
            catch (SQLException e) { e.printStackTrace(); }
        }
    }
    return -1;
}
```

### 5.4 商品分页查询与分类筛选 ProductDao.findAll()

```java
public List<Product> findAll(Integer categoryId, String keyword,
        int page, int pageSize) {
    StringBuilder sql = new StringBuilder(
        "SELECT p.*, c.name AS category_name FROM products p " +
        "LEFT JOIN categories c ON p.category_id = c.id WHERE 1=1"
    );
    List<Object> params = new ArrayList<>();

    if (categoryId != null && categoryId > 0) {
        sql.append(" AND p.category_id = ?");
        params.add(categoryId);
    }
    if (keyword != null && !keyword.trim().isEmpty()) {
        sql.append(" AND (p.name LIKE ? OR p.description LIKE ?)");
        params.add("%" + keyword.trim() + "%");
        params.add("%" + keyword.trim() + "%");
    }
    sql.append(" ORDER BY p.id DESC LIMIT ?, ?");
    params.add((page - 1) * pageSize);
    params.add(pageSize);

    List<Product> list = new ArrayList<>();
    Connection conn = null;
    PreparedStatement stmt = null;
    ResultSet rs = null;
    try {
        conn = DBUtil.getConnection();
        stmt = conn.prepareStatement(sql.toString());
        for (int i = 0; i < params.size(); i++) {
            stmt.setObject(i + 1, params.get(i));
        }
        rs = stmt.executeQuery();
        while (rs.next()) {
            list.add(mapRow(rs));
        }
    } catch (SQLException e) {
        e.printStackTrace();
    } finally {
        DBUtil.close(conn, stmt, rs);
    }
    return list;
}
```

```java
    public void doFilter(ServletRequest request, ServletResponse response,

    }
}

// AdminFilter.java — 管理员权限过滤
@WebFilter(urlPatterns = {"/admin/*"})
public class AdminFilter implements Filter {
    @Override
    public void doFilter(ServletRequest request, ServletResponse response,
            FilterChain chain) throws IOException, ServletException {
        HttpServletRequest req = (HttpServletRequest) request;
        HttpServletResponse resp = (HttpServletResponse) response;
        User user = (User) req.getSession().getAttribute("user");
        if (user == null) {
            resp.sendRedirect(req.getContextPath() + "/login.jsp");
            return;
        }
        if (!user.isAdmin()) {
            resp.setContentType("text/html;charset=UTF-8");
            resp.getWriter().write("<script>alert('权限不足！');history.back();</script>");
            return;
        }
        chain.doFilter(req, resp);
    }
}
```

### 7.2 调试过程中出现的错误及解决方法

#### 问题1：数据库连接失败——"Communications link failure"

**错误描述**：项目启动后，访问任何Servlet页面都报500错误，控制台输出 `com.mysql.cj.jdbc.exceptions.CommunicationsException: Communications link failure`。

**原因分析**：MySQL服务未启动，或db.properties中配置的数据库URL、用户名、密码不正确。

**解决方法及步骤**：
（1）检查MySQL服务是否已启动：在Windows服务列表中确认MySQL80服务状态为"正在运行"。
（2）验证数据库连接参数：确认db.properties中的db.url主机地址为localhost，端口为3306，数据库名为shopping，用户名和密码与MySQL安装时设置的保持一致。
（3）将在db.properties中配置而非硬编码的数据库连接信息分离出来，方便不同环境下的部署。

#### 问题2：中文乱码——页面显示?????

**错误描述**：JSP页面中从数据库读取的中文字段显示为问号或乱码，表单提交的中文数据写入数据库后也变成乱码。

**原因分析**：数据在"浏览器 → Tomcat → JDBC → MySQL"的传输链路中，某个环节的编码未设置为UTF-8。

**解决方法及步骤**：
（1）在MySQL中执行 `ALTER DATABASE shopping CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;` 确保数据库默认编码为utf8mb4。
（2）在JDBC连接URL中添加参数 `useUnicode=true&characterEncoding=utf-8&serverTimezone=Asia/Shanghai`。
（3）创建EncodingFilter全局过滤器，设置 `request.setCharacterEncoding("UTF-8")` 和 `response.setContentType("text/html;charset=UTF-8")`，并将其urlPatterns设为"/*"覆盖所有请求。
（4）在每个JSP页面头部添加 `<%@ page contentType="text/html;charset=UTF-8" language="java" %>` 指令。

#### 问题3：下单时库存扣减不一致

**错误描述**：用户下单时，偶尔出现订单已生成但库存未扣减，或库存扣减了但订单未生成的情况。

**原因分析**：下单操作涉及多张表的写入（orders、order_items、products库存更新、cart_items删除），之前代码中这些操作分布在多个独立的数据库连接中执行，没有使用事务管理，导致部分操作成功部分失败时数据不一致。

**解决方法及步骤**：
（1）将下单的四个操作（插入订单、插入明细、扣库存、清空购物车）统一放入一个数据库事务中。
（2）在获取连接后调用 `conn.setAutoCommit(false)` 关闭自动提交。
（3）所有操作成功执行后调用 `conn.commit()` 提交事务。
（4）在catch块中调用 `DBUtil.rollback(conn)` 回滚事务。
（5）在finally块中恢复 `conn.setAutoCommit(true)` 并关闭连接。

#### 问题4：购物车添加商品时重复创建记录

**错误描述**：用户多次将同一商品加入购物车后，购物车中出现多条同一商品的记录，而非数量叠加。

**原因分析**：CartDao中add()方法未检查该用户购物车中是否已存在该商品，直接执行INSERT语句。

**解决方法及步骤**：在add()方法中加入前置检查逻辑——先调用 `findByUserAndProduct(userId, productId)` 查询是否已存在，若存在则调用 `updateQuantity()` 将数量累加；若不存在才执行INSERT插入新记录。修改后同一商品在购物车中只存在一条记录，数量自动累加。

#### 问题5：未登录用户可直接访问购物车页面

**错误描述**：在地址栏直接输入 "/cart.jsp" 路径，未登录状态下也能看到购物车页面（虽然数据为空），并且页面出现空指针异常。

**原因分析**：购物车JSP页面在处理时尝试从Session中获取User对象，但未登录时user为null，导致后续代码调用 `user.getId()` 时抛出NullPointerException。

**解决方法及步骤**：
（1）创建LoginFilter过滤器，urlPatterns设为需要登录的页面路径（/cart.jsp、/orders.jsp等）。
（2）在doFilter方法中检查Session中的user属性是否为null，若为null则重定向到登录页面。
（3）在LoginServlet中增加redirectUrl记录功能，登录成功后自动跳回用户原本想访问的页面，提升用户体验。

### 8.1 项目中用到的知识点

通过本次大作业的开发实践，我综合运用了《JavaWeb开发技术》课程中的多项核心技术：

1. **Servlet技术**：使用HttpServlet处理GET/POST请求，通过@WebServlet注解进行URL映射，利用HttpSession管理用户会话状态。

2. **JSP与视图层技术**：使用JSP作为视图模板，结合JSTL标签库（`<c:forEach>`、`<c:if>`等）和EL表达式进行动态数据渲染。通过`<%@ include %>`指令复用header.jsp和footer.jsp公共组件，减少代码重复。

3. **JDBC数据库编程**：使用JDBC API进行数据库连接和操作，通过PreparedStatement预编译语句防止SQL注入，使用Statement.RETURN_GENERATED_KEYS获取自增主键，利用Connection.setAutoCommit(false)实现事务管理。

4. **过滤器（Filter）技术**：编写并配置了EncodingFilter（全局编码统一）、LoginFilter（登录状态拦截）、AdminFilter（管理员权限验证），理解了过滤链的执行顺序和作用范围。

5. **MVC分层架构**：将项目清晰地划分为Model层（实体类）、DAO层（数据访问）、Servlet层（控制器）、JSP层（视图），各层职责明确，便于维护和扩展。

6. **Maven项目管理**：使用Maven进行项目构建和依赖管理（servlet-api、jsp-api、jstl、mysql-connector等），配置tomcat7-maven-plugin实现一键运行。

7. **MySQL数据库操作**：掌握了数据库的创建、表结构设计、外键约束、多表连接查询（LEFT JOIN）、聚合函数（COUNT）、MD5哈希函数的使用。

1. **商品图片管理**：当前商品图片依赖手动放置图片文件到images目录，未实现图片上传功能。后续可引入文件上传组件实现管理员通过后台上传商品图片。

3. **支付功能**：当前订单状态由管理员手动修改，无实际支付接口。后续可对接支付宝或微信支付的沙箱环境实现模拟支付。

4. **输入验证**：当前前后端的输入验证不够完善（如邮箱格式、密码强度等），后续可增加正则表达式验证和JavaScript前端校验。

5. **分页优化**：当前分页每次查询都执行一次COUNT(*)统计总数，数据量较大时可能存在性能问题，可考虑缓存总数或使用覆盖索引。
